Thought process
Thought process
Boa ideia — é um projeto clássico de pipeline de voz. Eu o dividiria em 5 componentes encadeados:

1. Wake word ("Cláudio")
Detecção contínua e local (nunca envie áudio contínuo para a nuvem). Opções: Picovoice Porcupine (permite treinar palavra customizada como "Cláudio", ótimo desempenho, plano gratuito para uso pessoal) ou openWakeWord (open source). Alternativa nativa: o próprio Speech framework da Apple rodando local, filtrando pela palavra — menos eficiente, mas sem dependências.

2. Captura e fim de fala (VAD)
Após o wake word, gravar via AVAudioEngine e detectar quando você parou de falar (ex.: Silero VAD ou simples threshold de silêncio de ~1,5s) para encerrar a pergunta.

3. Speech-to-Text

Local: whisper.cpp (excelente em português, roda bem em Apple Silicon) ou SFSpeechRecognizer da Apple (grátis, nativo, suporta pt-BR on-device).
Nuvem: API do Whisper/Deepgram — menor esforço, mais latência e custo.
4. LLM
Enviar o texto à API do Claude (Messages API), mantendo histórico da conversa para contexto. Um system prompt pedindo respostas curtas e "faladas" melhora muito a experiência de voz.

5. Text-to-Speech

Nativo: AVSpeechSynthesizer (grátis, vozes pt-BR razoáveis, ex. Luciana).
Premium: ElevenLabs ou OpenAI TTS (voz muito mais natural, pago, com streaming para reduzir latência).
Estrutura do app
Menu bar app em Swift/SwiftUI (ícone indicando estados: dormindo / ouvindo / pensando / falando). Precisa de permissão de microfone (NSMicrophoneUsageDescription) e cuidado com echo cancellation para o assistente não "se ouvir" respondendo — o voiceProcessing mode do AVAudioEngine resolve.

Fases que eu seguiria

Protótipo em Python (dias): pipeline completo no terminal para validar wake word, STT em português e latência.
App Swift de menu bar com o pipeline portado.
Refinos: streaming da resposta (começar a falar antes do Claude terminar), interrupção ("Cláudio, para"), histórico.
Principais riscos: latência total (meta: <2s do fim da pergunta ao início da fala) e qualidade do wake word customizado em português.

Como funciona: com a chave definida, o app envia o texto da resposta para a ElevenLabs (eleven_multilingual_v2, o modelo de maior qualidade, com ótimo pt-BR) e toca o MP3 retornado. Sem a chave — ou se a API falhar — volta automaticamente para a voz do sistema. Seu áudio de microfone continua nunca saindo do Mac.

Ajustes opcionais: ELEVENLABS_VOICE_ID para escolher outra voz (o padrão é a Rachel; o README mostra como listar as suas) e ELEVENLABS_MODEL=eleven_flash_v2_5 se preferir respostas mais rápidas em troca de um pouco de qualidade.

Nota: a ElevenLabs tem plano gratuito (~10 mil créditos/mês); acima disso é pago por caractere.
